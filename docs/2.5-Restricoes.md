# 2.5 Restrições

As restrições representam limitações e condições que influenciam o desenvolvimento e a implantação do sistema AgiliCar.

Esses fatores devem ser considerados durante todo o ciclo do projeto, pois definem aspectos de escopo, tecnologia, prazo e integrações disponíveis para a solução.

## Restrições

O projeto está restrito pelos itens apresentados na tabela a seguir.

| ID  | Restrição                                                                                             |
| --- | ----------------------------------------------------------------------------------------------------- |
| 01  | O projeto deverá ser entregue até o final do 1º semestre de 2026                                      |
| 02  | Não haverá integração com sistemas externos de pagamento ou ERP nesta versão                          |
| 03  | O sistema requer conexão com a internet e um servidor local de banco de dados MySQL                   |
| 04  | As notificações ao cliente serão registradas no sistema, sem envio real por e-mail ou SMS             |
| 05  | O módulo de assinatura digital será representado por confirmação de entrega no sistema                |
| 06  | O registro de pagamentos será realizado apenas de forma interna, sem processamento financeiro externo |
