# 2.2 Histórias de Usuário

As histórias de usuário descrevem as necessidades dos usuários do sistema sob sua própria perspectiva, permitindo identificar funcionalidades que agregam valor aos processos da oficina.

Elas foram elaboradas com base nas personas definidas anteriormente e seguem o formato:

> Eu como [usuário], quero [funcionalidade], para [objetivo].

As histórias apresentadas servem como base para a definição dos requisitos funcionais do sistema AgiliCar.
