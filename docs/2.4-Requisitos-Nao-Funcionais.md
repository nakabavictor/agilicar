# 2.4 Requisitos Não Funcionais

Os requisitos não funcionais definem características de qualidade que devem ser atendidas pelo sistema AgiliCar, como segurança, desempenho, usabilidade e infraestrutura tecnológica.

Embora não representem funcionalidades diretamente visíveis ao usuário, esses requisitos são fundamentais para garantir confiabilidade, eficiência e uma boa experiência de utilização da solução.

## Requisitos Não Funcionais

| ID      | Descrição do Requisito                                                           | Prioridade |
| ------- | -------------------------------------------------------------------------------- | ---------- |
| RNF-001 | O sistema deve ser acessível via navegador web sem necessidade de instalação     | ALTA       |
| RNF-002 | A autenticação deve utilizar tokens JWT com expiração de 8 horas                 | ALTA       |
| RNF-003 | As senhas dos usuários devem ser armazenadas com hash BCrypt                     | ALTA       |
| RNF-004 | O controle de acesso deve ser baseado em perfis (Gestor, Recepcionista, Técnico) | ALTA       |
| RNF-005 | O sistema deve processar as requisições do usuário em no máximo 3 segundos       | MÉDIA      |
| RNF-006 | A interface deve ser intuitiva e utilizável sem treinamento extenso              | MÉDIA      |
| RNF-007 | Os dados devem ser persistidos em banco de dados relacional MySQL                | ALTA       |
