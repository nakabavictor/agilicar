# 2. Especificação do Projeto

**Pré-requisito:** [1. Contexto](./1-Contexto.md)

Esta seção apresenta a especificação do sistema **AgiliCar**, contemplando os perfis dos usuários, suas necessidades, histórias de usuário, requisitos funcionais, requisitos não funcionais e restrições do projeto.

O objetivo é detalhar as funcionalidades e características da solução proposta, garantindo o alinhamento entre os processos de negócio identificados e o sistema a ser desenvolvido.

## Estrutura da Especificação

Os artefatos desta etapa estão organizados nos documentos abaixo:

### 2.1 Personas

Descrição dos perfis de usuários que utilizarão o sistema.

- [Acessar documento](./2.1-Personas.md)

### 2.2 Histórias de Usuário

Levantamento das necessidades dos usuários utilizando o formato de histórias de usuário.

- [Acessar documento](./2.2-Historias-de-Usuario.md)

### 2.3 Requisitos Funcionais

Funcionalidades que o sistema deve oferecer para atender às necessidades do negócio.

- [Acessar documento](./2.3-Requisitos-Funcionais.md)

### 2.4 Requisitos Não Funcionais

Características de qualidade, desempenho, segurança e usabilidade da solução.

- [Acessar documento](./2.4-Requisitos-Nao-Funcionais.md)

### 2.5 Restrições

Limitações técnicas, operacionais e de escopo consideradas durante o desenvolvimento do projeto.

- [Acessar documento](./2.5-Restricoes.md)
